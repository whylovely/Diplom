## 🧪 Ручное тестирование синхронизации

### 1. Подготовка

1. Запустить сервер: `cd Server && dotnet run`
2. Убедиться, что сервер доступен: `http://localhost:5273/swagger`
3. Запустить клиент: `cd Client && dotnet run`

---

### 2. Тест: Базовая синхронизация

1. Открыть клиент, войти как `demo@finance.local` / `Demo123`
2. Нажать кнопку **🔄 Синхронизировать** в окне «Счета»
3. ✅ Ожидается: статус сменяется на «Синхронизировано», счета загружены

---

### 3. Тест: Отчёты после синхронизации

1. Синхронизировать данные (шаг 2)
2. Перейти в раздел **Отчёты**
3. ✅ Ожидается: все вкладки (Расходы, Доходы, Баланс, Счета) показывают данные

---

### 4. Тест: Категории после синхронизации

1. Синхронизировать данные
2. Перейти в раздел **Категории**
3. ✅ Ожидается: список категорий не пустой

---

### 5. Тест: Удаление счёта (К44)

| Сценарий | Действие | Ожидаемый результат |
|----------|----------|---------------------|
| Баланс ≠ 0 | Выбрать счёт с балансом, нажать «Удалить» | ❌ Ошибка: «Обнулите баланс» |
| Баланс = 0, есть операции | Нажать «Удалить» | ⚠️ Предупреждение + подтверждение |
| Баланс = 0, нет операций | Нажать «Удалить» | ✅ Счёт удалён |

---

### 6. Тест: Ошибка подключения

1. Остановить сервер (`Ctrl+C`)
2. Нажать **Синхронизировать** в клиенте
3. ✅ Ожидается: статус красный, сообщение об ошибке (без краша)

### 7. Тест: К45 — Диалог конфликта синхронизации

**Подготовка:** данных на клиенте должно быть другое количество, чем на сервере.

| Сценарий | Действие | Ожидаемый результат |
|----------|----------|---------------------|
| Клиент и сервер совпадают | Синхронизировать | Диалог НЕ появляется, синхронизация тихая |
| Данные различаются → **«Взять с сервера»** | Выбрать в диалоге | Локальные данные заменяются серверными |
| Данные различаются → **«Оставить мои»** | Выбрать в диалоге | Данные не меняются, статус — «Отменено» |
| Данные различаются → **«Отмена»** | Закрыть диалог | Данные не меняются, статус остаётся |

> **Как симулировать конфликт**: добавить транзакцию вручную локально, затем нажать синхронизировать — на сервере другое количество.

**Изменения (только клиент, сервер не менялся):**
- `SyncConflictDialog.axaml` — новый диалог с выбором
- `SyncService.GetServerTransactionCountAsync()` — счёт операций с сервера
- `LocalDbService.GetLocalTransactionCount()` / `GetLocalLastChangeDate()` — локальные данные
- `AccountsViewModel.SyncAsync()` — сравнение и показ диалога
