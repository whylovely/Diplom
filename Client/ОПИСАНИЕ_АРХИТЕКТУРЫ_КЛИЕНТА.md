# Архитектура клиентского приложения

Клиент — десктопное приложение для учёта личных финансов. Пользователь ведёт счета, записывает доходы и расходы, смотрит отчёты и управляет долгами.

---

## 1. Технологии

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| Язык | C# 12 (.NET 8) | Основной язык |
| UI-фреймворк | Avalonia UI 11.3 | Кроссплатформенный GUI (Windows, macOS, Linux) |
| Архитектура | MVVM | Разделение данных, логики и интерфейса |
| Кодогенерация MVVM | CommunityToolkit.Mvvm 8.2 | `[ObservableProperty]`, `[RelayCommand]` |
| Визуальная тема | Avalonia.Themes.Fluent (Dark) | Тёмная тема в стиле Windows 11 |
| Таблицы | Avalonia.Controls.DataGrid | Компонент таблиц для списков |
| Графики | LiveCharts2 (SkiaSharp) | Диаграммы и графики в отчётах |
| Экспорт в Excel | ClosedXML | Формирование .xlsx файлов |
| Локальная БД | SQLite (Microsoft.Data.Sqlite) | Хранение данных офлайн |
| Шрифт | Inter (Avalonia.Fonts.Inter) | Основной шрифт приложения |

## 2. Структура папок

```
Client/
├── Models/            — Классы данных (Account, Transaction, Category, ...)
├── Views/             — AXAML-разметка интерфейса
│   └── DialogViews/   — Всплывающие диалоговые окна
├── ViewModels/        — Логика интерфейса (MVVM)
│   ├── DialogWindow/  — ViewModels для диалогов
│   └── OperationWithReport/ — Логика экспорта отчётов
├── Services/          — Сервисы (данные, настройки, уведомления, синхронизация)
├── Assets/            — Ресурсы (иконки, шрифты)
├── Program.cs         — Точка входа
├── App.axaml          — Глобальные стили и DataTemplates
├── App.axaml.cs       — Инициализация, обработка глобальных ошибок
└── Client.csproj      — Файл проекта (зависимости)
```

## 3. Запуск приложения (пошагово)

### Шаг 1: `Program.cs`

```csharp
[STAThread]
public static void Main(string[] args) => BuildAvaloniaApp()
    .StartWithClassicDesktopLifetime(args);
```

Создаёт движок Avalonia (тема, шрифты, платформа) и запускает десктопное окно.

### Шаг 2: `App.axaml.cs`

```csharp
public override void OnFrameworkInitializationCompleted()
{
    var win = new MainWindow { DataContext = new MainWindowViewModel() };
    desktop.MainWindow = win;
}
```

Создаёт главное окно и привязывает к нему `MainWindowViewModel`. Также вешает глобальные обработчики исключений (`UnhandledException`, `UnobservedTaskException`), которые показывают ошибку через `MessageDialog`.

### Шаг 3: `MainWindowViewModel`

В конструкторе создаёт все сервисы и все страницы приложения:

```csharp
// Сервисы
_data = new LocalDbService();          // Локальная SQLite БД
_notify = new NotificationService();   // Уведомления (диалоги)
_settings = new SettingsService();     // Настройки (JSON-файл)
_auth = new AuthService(_settings);    // JWT-авторизация
_rateService = new CurrencyRateService(_settings); // Курсы валют
var apiService = new ApiService(_settings);        // HTTP-клиент к серверу
var syncService = new SyncService(apiService, (LocalDbService)_data); // Синхронизация

// Страницы (ViewModels)
DashboardVm, AccountsVm, JournalVm, NewTxVm, ReportVm,
CategoriesVm, ObligationsVm, SettingsVm, CurrenciesVm
```

Стартовая страница — `DashboardVm`.

### Шаг 4: `OnWindowLoaded()`

При загрузке окна выполняется цепочка:
1. Проверка авторизации → если нет — диалог логина
2. Первый запуск → диалог выбора базовой валюты
3. Обновление курсов валют из интернета
4. Проверка просроченных/подходящих обязательств → уведомление

## 4. Навигация между страницами

В `MainWindow.axaml` — `ContentControl` с привязкой к `Current`:

```xml
<ContentControl Content="{Binding Current}" />
```

Навигация через команды:

```csharp
[RelayCommand] private void NavigateAccounts() { Current = AccountsVm; ActivePage = "Accounts"; }
```

`DataTemplates` в `MainWindow.axaml` связывают ViewModel → View:

```xml
<DataTemplate DataType="vm:AccountsViewModel">
  <views:AccountsView/>
</DataTemplate>
```

Внизу экрана — панель навигации (стилизованная под macOS Dock).

## 5. Модели данных

| Модель | Назначение |
|--------|------------|
| `Account` | Счёт (название, валюта, баланс, тип, мультивалютность) |
| `Transaction` | Финансовая операция (дата, описание, список проводок) |
| `Entry` | Одна проводка (счёт, категория, направление, сумма) |
| `Category` | Категория расходов/доходов |
| `Obligation` | Долг/обязательство (контрагент, сумма, срок) |
| `AccountGroup` | Группировка счетов по валюте/типу |
| `CurrencyRate` | Курс валюты к базовой |
| `TransactionTemplate` | Шаблон частой операции |
| `CalendarDay` | Информация о транзакциях за день (для календаря) |
| `JournalRow` | Строка журнала операций |
| `MonthlyTotalRow` | Месячный итог (доходы/расходы) |
| `Money` | Сумма + валюта |
| `UserSettings` | Настройки (базовая валюта, первый запуск) |
| `MessageLevel` | Уровень сообщения (Info, Warning, Error) |

## 6. Сервисы

### `IDataService` / `LocalDbService`
Интерфейс + реализация хранилища данных. `LocalDbService` работает с SQLite-файлом `finance.db`. Хранит счета, категории, транзакции, обязательства. Поддерживает двойную запись, сторнирование, пересчёт балансов.

### `ApiService`
HTTP-клиент для REST API сервера. Отправляет запросы с JWT-токеном в заголовке.

### `SyncService`
Двусторонняя синхронизация: Pull (сервер → клиент) и Push (клиент → сервер). При конфликте — показывает `SyncConflictDialog`.

### `AuthService`
JWT-авторизация: логин, регистрация, хранение токена, проверка `IsLoggedIn`.

### `CurrencyRateService`
Загружает курсы валют с сервера и кэширует в `LocalDbService`.

### `SettingsService`
Сохраняет/читает настройки из JSON-файла (`user_settings.json`): базовая валюта, первый запуск, URL сервера.

### `NotificationService`
Показывает диалоговые окна: ошибки, предупреждения, информация, подтверждение.

### `CategoryDialogService` / `InputDialogService`
Диалоги для выбора/создания категорий и ввода текста.

## 7. ViewModels (страницы)

| ViewModel | Страница | Назначение |
|-----------|----------|------------|
| `MainWindowViewModel` | Главное окно | Навигация, инициализация всех сервисов и страниц |
| `DashboardViewModel` | Главный экран | Сводка: баланс, последние операции, топ расходов, график |
| `AccountsViewModel` | Счета | CRUD счетов, группировка, быстрый переход к транзакции |
| `CategoriesViewModel` | Категории | Добавление, переименование, удаление |
| `JournalViewModel` | Журнал | Таблица всех транзакций с поиском и цветовой подсветкой |
| `NewTransactionViewModel` | Новая транзакция | Форма: расход, доход, перевод; шаблоны операций |
| `ReportViewModel` | Отчёты | Графики, таблицы, экспорт (Excel/CSV/TXT) |
| `ObligationsViewModel` | Обязательства | Карточки долгов, отметка оплаты, создание транзакции |
| `CurrenciesViewModel` | Курсы валют | Просмотр курсов, управление избранными валютами |
| `SettingsViewModel` | Настройки | Базовая валюта, синхронизация, выход из аккаунта |
| `CalendarViewModel` | Календарь | Операции по дням в виде календарной сетки |
