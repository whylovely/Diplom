# Дизайн-система и UI Kit клиента (Avalonia UI)

Этот документ является профессиональным руководством для разработчиков (Design Guidelines) клиентского финансового приложения, реализованного на Avalonia UI. 

Дизайн-система ориентирована на современный премиальный темный интерфейс (Dark Mode) с эффектами матового стекла (Acrylic), геометрической типографикой, цветным свечением и продвинутыми микровзаимодействиями. Вдохновением послужили стилистика macOS (док-панель навигации, размытие) и современные футуристичные финтех-продукты. 

---

## 1. Общая концепция и макет (Layout & Navigation)
- **Тематика**: Глубокая темная тема с сине-фиолетовыми неоновыми акцентами.
- **Стиль**: Чистый, минималистичный, «Soft UI» с использованием закругленных карточек.
- **Базовая тема Avalonia**: `FluentTheme`.
- **Главный контейнер**: Приложение не имеет стандартного верхнего или бокового системного меню. `MainWindow` делится на две логические части:
  1. Рабочая область сверху (с отступами `24, 20, 24, 10`).
  2. Плавающая нижняя навигационная панель (Dock).
- **Плавающий Dock**:
  - Находится снизу по центру экрана.
  - Имеет сильное скругление углов (`CornerRadius="32"`), легкую границу и отбрасывает эффект размытия позади себя (`ExperimentalAcrylicBorder` с `TintColor="#0A0E1A"`, `TintOpacity="0.2"`).
  - Центральная кнопка навигации ("Новая операция") выделяется ярким синим фоном.

## 2. Цветовая палитра (Color Tokens)

### 2.1 Базовые фоны и поверхности (Backgrounds & Surfaces)
- **Главный фон окна**: `#080C1A` (Глубокий темно-синий/почти черный).
- **Фон панелей и карточек**: `#B0111528` (Полупрозрачный синеватый), `#0D1120`, `#151A2E`.
- **Фон таблиц (DataGrid)**: `#0A0E1E`.
- **Линии и границы (Borders)**: `#222845`.

### 2.2 Акцентные цвета и градиенты
- **Основной акцент (Кнопки, формы)**: `#1E88C8` (Яркий синий).
- **Акцентный Hover**: `#1E88C820` (Полупрозрачный синий).
- **Linear Gradients (Action & Hero)**:
  - Основной градиент: от `#29B6F6` (Светло-голубой) до `#AA00FF` (Фиолетовый).
  - Hover градиент: от `#4FC3F7` до `#CE93D8`.

### 2.3 Финансовые статусы
- **Доход (Позитив)**: `#00E676` (Ярко-зеленый) — Класс `TextBlock.moneyGreen`.
- **Расход (Опасность)**: `#FF5252` (Ярко-красный) — Класс `TextBlock.moneyRed`.

### 2.4 Палитра категорий (Для Donut Chart / Графиков)
Для отображения категорий расходов используются пастельно-неоновые оттенки:
- **Транспорт**: `#80D8FF`
- **Развлечения**: `#EA80FC`
- **Продукты**: `#FF9E80`
- **Дом/ЖКХ**: `#B9F6CA`
- **Переводы/Прочее**: `#FFE57F`

## 3. Типографика (Typography)

Строгое разделение шрифтов для обеспечения премиального вида текстов и инженерного вида финансовых показателей.

### 3.1 Шрифтовые гарнитуры
- **Основной шрифт (Тексты, интерфейс)**: `Plus Jakarta Sans`. Придает приложению современный, премиальный и геометрический вид. Фоллбэк: `Segoe UI, sans-serif`.
- **Шрифт для цифр и сумм (Балансы, DataGrid)**: `Fira Code`. Применяется **исключительно** для отображения цифр, дат и валют, придавая им моноширинный технический вид.

### 3.2 Техническое правило Avalonia (OpenType Features)
Для всех финансовых сумм в `DataGrid` и списках **строго обязательно** использование типографических OpenType-фич. Это гарантирует ровное выравнивание колонок сумм.

```xml
<!-- Пример применения Fira Code для сумм -->
<TextBlock FontFamily="Fira Code" 
           Text="10 500,00 ₽" 
           Typography.NumeralAlignment="Tabular" 
           Typography.SlashedZero="True" />
```

### 3.3 Классы текста
- `heading`: Размер 24, Bold, `#FFFFFF`. Для заголовков страниц.
- `subheading`: Размер 16, SemiBold, `#E0E0E0`. Для подзаголовков карточек.
- `dim`: Размер 12, `#9E9E9E`. Вспомогательные данные.
- `cyan`: Акцентный текст (`#29B6F6`), Bold.

## 4. Компоненты и Стилизация (Glow vs Shadows)

**Строгое правило: ЗАПРЕЩАЕТСЯ использование стандартных черных теней (`DropShadow`).** 
Проект использует эффект цветного неонового свечения (Glow) для активных элементов через свойство `BoxShadow`.

### 4.1 Цветное свечение (BoxShadow Glow)
Используется для выделения кнопок, активных карточек или фокуса:

```xml
<Border Background="#151A2E" 
        CornerRadius="16" 
        BoxShadow="0 4 20 0 #2029B6F6">
    <!-- Содержимое с голубым свечением -->
</Border>
```

### 4.2 Hero-секция (Главный дашборд)
Общий баланс пользователя должен являться центральным элементом внимания. Для заливки суммы используется линейный градиент.

```xml
<TextBlock FontFamily="Fira Code" 
           FontSize="48" FontWeight="Bold"
           Typography.SlashedZero="True" 
           Typography.NumeralAlignment="Tabular">
    <TextBlock.Foreground>
        <LinearGradientBrush StartPoint="0%,0%" EndPoint="100%,100%">
            <GradientStop Color="#29B6F6" Offset="0" />
            <GradientStop Color="#AA00FF" Offset="1" />
        </LinearGradientBrush>
    </TextBlock.Foreground>
    2 450 000.00
</TextBlock>
```

### 4.3 Кнопки (Buttons)
1. **Стандартная**: Фон `#151A2E`, скругление `8px`. При наведении фон `#1E88C8`, текст `#FFFFFF`.
2. **Экстра-акцентная (.accent)**: Градиентная кнопка (от `#29B6F6` к `#AA00FF`) для главных действий.
3. **Danger-кнопка (.danger)**: Чтобы не сливаться с глубоким фоном, текст должен быть всегда контрастным. В спокойном состоянии текст и граница яркие `#FF5252`, фон темный `#1F0F1C`. При наведении кнопка заливается красным полностью.
4. **Dock-кнопки**: Прозрачные, `CornerRadius="24"`. Иконка векторная (`Path Data`).

## 5. Состояния UI (Загрузка и Пустые экраны)

### 5.1 Скелетная загрузка (Skeleton Loading)
Приложение отказывается от классических круглых спиннеров в пользу UI-скелетов. 
Во время загрузки данных из БД (сети или SQLite), показывать мерцающие заглушки:
- **Цвет**: `#151A2E`.
- **Анимация**: Плавное изменение прозрачности (Opacity от `0.3` до `1.0`).

### 5.2 Empty States (Пустые состояния)
Если в таблице или списке нет данных, DataGrid скрывается и отображается визуально приятная заглушка:
- Крупная векторная иллюстрация (Opacity `10-15%`).
- Приглушенный текст (`#9E9E9E`, например «Здесь пока нет операций...»).

## 6. Интерактивность и Микро-взаимодействия

### 6.1 Индикация синхронизации (PostgreSQL / Локальная БД)
В углу окна (либо рядом с заголовком приложения) располагается компактный индикатор статуса (иконка облака или круговых стрелок):
- 🟢 **Синхронизировано** — Цвет иконки `#9E9E9E` (Не привлекает внимание).
- 🟡 **Локальные изменения / Ожидание сети** — Желтый / Оранжевый оттенок.
- 🔴 **Ошибка синхронизации** — Красный цвет.

### 6.2 Микро-взаимодействия
Добавление мелких, но премиальных деталей отклика UI:
- **Копирование данных**: При клике на номер счета/сумму для копирования, текст мгновенно мигает ярко-зеленым цветом (`#00E676`) на долю секунды, индицируя успешное помещение в буфер обмена.
- **Успешная отправка данных**: В момент успешного коммита транзакции в облако (или по завершению Sync), на индикаторе статуса должен пробегать короткий пульсирующий градиентный блик.

---

## 7. Иконки
- **Формат**: Исключительно векторные пути XAML (`<Path Data="..." />`).
- **Стилизация**: Цвет иконок наследуется от свойства `Foreground` родителя или привязывается динамически через `Fill`. Растровые изображения (PNG, JPG) для интерфейсных деталей не допускаются.
