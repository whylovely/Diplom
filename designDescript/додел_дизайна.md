Действуй как Senior UI/UX Designer и эксперт по разработке на Avalonia UI. У меня есть базовое описание дизайн-системы для клиентского финансового приложения (Dark Mode, macOS style, эффекты матового стекла).

Твоя задача — полностью обновить и расширить этот Markdown-документ, органично интегрировав в него список новых требований и технических правок. Документ должен выглядеть как профессиональное руководство для разработчиков (UI Kit & Design Guidelines).

Внеси следующие изменения и дополнения в соответствующие разделы документа:

1. Типографика и шрифты:

Замени основной шрифт с Inter на Plus Jakarta Sans (для придания премиального, геометрического вида).

Введи дополнительный шрифт Fira Code исключительно для отображения цифр, сумм и балансов (чтобы придать инженерный вид).

Добавь строгое техническое правило для Avalonia: для всех сумм в DataGrid и списках обязательно использование OpenType-фич Typography.NumeralAlignment="Tabular" (моноширинные цифры) и Typography.SlashedZero="True" (перечеркнутый ноль для футуристичности).

2. Цвета и градиенты:

Создай новый подраздел «Палитра категорий (Donut Chart)» с пастельно-неоновыми цветами: Транспорт (#80D8FF), Развлечения (#EA80FC), Продукты (#FF9E80), Дом/ЖКХ (#B9F6CA), Переводы/Прочее (#FFE57F).

Обнови кнопку «Danger»: текст должен быть ярким #FF5252 в спокойном состоянии (или иметь красную границу Border), чтобы не сливаться с темным фоном #1F0F1C.

Добавь правило для Hero-секции (Главного дашборда): общий баланс должен быть залит линейным градиентом от циана (#29B6F6) к фиолетовому (#AA00FF) через LinearGradientBrush.

3. Компоненты и стилизация (Glow vs Shadows):

Запрети использование стандартных черных теней (DropShadow). Введи правило «Цветного свечения» через свойство BoxShadow у Border (например, BoxShadow="0 4 20 0 #2029B6F6" для активных элементов).

4. Состояния UI (Загрузка и Пустые экраны):

Добавь подраздел «Скелетная загрузка (Skeleton Loading)»: вместо спиннеров использовать мерцающие заглушки цвета #151A2E с анимацией прозрачности.

Добавь подраздел «Empty States (Пустые состояния)»: для пустых экранов использовать крупные полупрозрачные иллюстрации (Opacity 10-15%) и приглушенный текст #9E9E9E (например, «Здесь пока нет операций...»).

5. Индикация и Микро-взаимодействия:

Опиши логику индикатора синхронизации с PostgreSQL: маленькая иконка (облако/стрелки) в углу или у заголовка. Цвета: #9E9E9E (Синхронизировано), желтый/оранжевый (Локальные изменения/Ожидание), красный (Ошибка).

Добавь подраздел «Микро-взаимодействия»:

Копирование данных: мигание текста зеленым (#00E676) на долю секунды.

Синхронизация БД: короткий пульсирующий блик градиентом на иконке статуса при успешной отправке данных.

Напиши итоговый Markdown-документ чисто, структурированно, сохраняя логические блоки (Общая концепция, Цвета, Типографика, Компоненты, Макет, Состояния и Интерактивность). Добавь примеры XAML-кода там, где это необходимо (например, для шрифтов и теней).
