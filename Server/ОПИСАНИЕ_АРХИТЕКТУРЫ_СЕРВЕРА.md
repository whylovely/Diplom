# Архитектура серверной части

Сервер — REST API, которое обслуживает запросы от клиентского приложения и веб-интерфейса администратора.

---

## 1. Технологии

| Компонент | Технология | Назначение |
|-----------|------------|------------|
| Язык | C# 12 (.NET 8) | Основной язык |
| Веб-фреймворк | ASP.NET Core 8 | HTTP-сервер, маршрутизация, middleware |
| ORM | Entity Framework Core 8 | Работа с БД через C#-классы |
| БД | PostgreSQL 16 | Хранение данных (Docker / облако) |
| Аутентификация | JWT (HS256) | Токены доступа |
| Хэширование паролей | ASP.NET Identity PasswordHasher | Безопасное хранение паролей |
| API документация | Swagger (Swashbuckle) | Тестирование API в браузере |
| Курсы фиатных валют | API ЦБ РФ (XML, windows-1251) | USD, EUR и т.д. к рублю |
| Курсы криптовалют | CoinGecko API (JSON) | BTC, ETH, USDT |
| Кэширование | IMemoryCache | 1 ч основной + 7 дней fallback |
| Кодировки | System.Text.Encoding.CodePages | Парсинг XML от ЦБ (windows-1251) |

## 3. Запуск сервера (`Program.cs`)

### 3.1. Регистрация сервисов

```csharp
builder.Services.AddControllers();             // Контроллеры
builder.Services.AddHttpClient();              // HTTP-клиент для внешних API
builder.Services.AddMemoryCache();             // Кэш курсов
builder.Services.AddScoped<IExchangeRateService, CbrExchangeRateService>();
builder.Services.AddDbContext<AppDbContext>(opt => opt.UseNpgsql(connectionString));
```

### 3.2. Строка подключения

Поддерживает два формата:
- Стандартный Npgsql: `Host=localhost;Port=5432;Database=finance;...`
- URI (для Render.com): `postgres://user:pass@host:port/db` — автоматически парсится в Npgsql-формат

### 3.3. JWT-аутентификация

```csharp
builder.Services
    .AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(opt => {
        // Issuer, Audience, SigningKey из appsettings.json
        // ClockSkew = 2 мин, RoleClaimType = ClaimTypes.Role
    });
builder.Services.AddAuthorization(options =>
    options.AddPolicy("Admin", policy => policy.RequireRole("Admin")));
```

### 3.4. Seed (начальные данные)

При запуске сервер автоматически:
1. Применяет EF Core миграции (`db.Database.Migrate()`)
2. Создаёт администратора из `appsettings.json` (admin@finance.local / Admin123)
3. Создаёт демо-пользователя (demo@finance.local / Demo123) с набором данных: 3 счёта, 5 категорий, 10 транзакций, 2 обязательства

### 3.5. Middleware-конвейер

```
Swagger → StaticFiles → Authentication → Authorization → MapControllers
```

Порядок важен: сначала проверяем кто (Authentication), потом имеет ли право (Authorization), потом обрабатываем запрос (Controllers).

## 4. Сущности (таблицы в БД)

| Сущность | Таблица | Ключевые поля |
|----------|---------|---------------|
| `UserEntity` | Users | Email, PasswordHash, Role (User/Admin), IsBlocked |
| `AccountEntity` | Accounts | Name, Kind, Currency, AccountType, SecondaryCurrency, ExchangeRate, IsDeleted |
| `CategoryEntity` | Categories | Name, IsDeleted |
| `TransactionEntity` | Transactions | Date, Description, Entries (навигация) |
| `EntryEntity` | Entries | AccountId, CategoryId, Direction (Debit/Credit), Amount, Currency |
| `ObligationEntity` | Obligations | Counterparty, Amount, Currency, Type (Debt/Credit), DueDate, IsPaid, IsDeleted |

Все сущности привязаны к пользователю через `UserId`. Soft delete: `IsDeleted` + глобальный фильтр в `AppDbContext`.

### Двойная запись

Каждая транзакция состоит из проводок (Entry). Пример — «Расход 1000 руб. на продукты»:
- Entry 1: Credit 1000 руб. со счёта «Наличные» (деньги уходят)
- Entry 2: Debit 1000 руб. на категорию «Продукты» (расход фиксируется)

## 5. Контроллеры (API)

| Контроллер | URL | Методы | Доступ |
|------------|-----|--------|--------|
| `AuthController` | `/api/auth` | POST register, POST login, POST admin/login | Публичный |
| `AccountsController` | `/api/accounts` | GET, GET/{id}, POST, PUT/{id}, DELETE/{id} | JWT |
| `CategoriesController` | `/api/categories` | GET, GET/{id}, POST, PUT/{id}, DELETE/{id} | JWT |
| `TransactionsController` | `/api/transactions` | GET, POST, DELETE/{id} | JWT |
| `ObligationsController` | `/api/obligations` | GET, POST, PUT/{id}, DELETE/{id}, POST/{id}/pay | JWT |
| `ReportController` | `/api/reports` | GET summary, by-category, monthly, turnover | JWT |
| `ExchangeController` | `/api/exchange` | GET rates | JWT |
| `SyncController` | `/api/sync` | GET pull, POST push | JWT |
| `AdminController` | `/api/admin` | GET users, POST block, DELETE user | Admin |

- `[Authorize]` — требует JWT-токен (401 без него)
- `[Authorize(Policy = "Admin")]` — только роль Admin (403 для User)
- `UserContext.GetUserId(User)` — извлекает ID из токена, фильтрует данные по пользователю

## 6. Сервис курсов валют

Два источника:
- **ЦБ РФ** — фиатные валюты (XML, `cbr.ru/scripts/XML_daily.asp`)
- **CoinGecko** — крипто (JSON, BTC/ETH/USDT → USD → пересчёт через курс доллара ЦБ)

Двухуровневый кэш:
1. Основной (IMemoryCache, TTL 1 час) — свежие данные
2. Fallback (TTL 7 дней) — отдаётся при ошибке внешнего API

## 7. Админ-панель

- API: просмотр пользователей, блокировка, удаление (`AdminController`)
- Веб-интерфейс: `wwwroot/admin/index.html` — статический HTML/JS, тёмная тема
- Вход через `POST /api/auth/admin/login`

## 9. Миграции

```bash
dotnet ef migrations add MigrationName   # Создать миграцию
dotnet ef database update                # Применить
```

Текущие миграции:
1. `InitialCreate` — базовые таблицы (Accounts, Transactions, Entries)
2. `AddUsersAndCategories` — пользователи и категории
3. `AddMultiCurrencyFields` — мультивалютные поля в счетах
4. `AddObligation` — таблица обязательств
5. `AddIsBlockedToUsers` — поле блокировки пользователей
