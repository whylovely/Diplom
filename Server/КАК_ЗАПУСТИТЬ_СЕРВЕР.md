# Инструкция: Как запустить и протестировать Сервер

Эта локальная серверная часть написана на **ASP.NET Core (C#)** и предоставляет REST API для работы клиентского приложения.

В процессе разработки для тестирования всех endpoints (точек доступа) мы используем встроенную автодокументацию и интерфейс **Swagger**.

## 🚀 1. Как запустить сервер

### Через консоль / терминал (предпочтительный способ)

1. Откройте терминал (PowerShell или Командную строку) в корневой папке вашего диплома.
2. Перейдите в папку с сервером:
   ```bash
   cd Server
   ```
3. Запустите проект командой:
   ```bash
   dotnet run
   ```
4. В консоли появятся строки о том, что сервер успешно запущен и прослушивает порты (обычно `http://localhost:5000` и `https://localhost:5001`).

### Через IDE (Visual Studio / Rider)

1. Откройте решение (или саму папку `Server`) в вашей среде разработки.
2. Убедитесь, что запускаемый проект выбран как `Server` (а не Client).
3. Нажмите кнопку **Run (Запуск)** — зеленый треугольник, или нажмите `F5` для запуска с отладкой.

## 🧪 2. Как протестировать API (Swagger)

Сразу после того как сервер запущен, вы можете отправлять запросы в базу данных напрямую из браузера без клиентского приложения.

1. Откройте любой браузер (Chrome, Edge, Safari и т.д.).
2. Перейдите по следующей ссылке (убедитесь, что проверяете правильный порт из консоли, чаще всего это 5001 для HTTPS или 5000 для HTTP):
   
   👉 **https://localhost:5001/swagger**
   
   *(Если порт другой, замените 5001 на тот, что написан в консоли при запуске сервера).*

### Как пользоваться Swagger UI:

Перед вами откроется веб-страница со списком всех доступных контроллеров (Endpoints) — таких как `Auth`, `Accounts`, `Categories`, `Transactions` и т.д.

1. **Разверните любой метод** (например, `GET /api/accounts`).
2. Нажмите кнопку **"Try it out"** в правом верхнем углу блока.
3. Если метод требует параметры (например ID для удаления), заполните их.
4. Нажмите большую синюю кнопку **"Execute"**.
5. Чуть ниже (в блоке *Responses*) вы увидите **Server response** — код ответа (например, 200 OK) и тело ответа с данными в формате JSON.

## 🔒 3. Как получить доступ к защищенным методам (Авторизация)

Некоторые эндпоинты защищены с помощью JWT токенов (например, редактирование счетов после регистрации). Чтобы выполнить их в Swagger:

1. Сначала найдите метод **`POST /api/auth/login`** (или `/api/auth/register`, если у вас еще нет пользователя).
2. Нажмите **Try it out**, введите логин и пароль в JSON теле, и нажмите **Execute**.
3. В успешном ответе (код 200) найдите поле `"token"`. Скопируйте длинную строку токена (без кавычек).
4. Поднимитесь в самый верх страницы Swagger и нажмите кнопку **"Authorize"** (иконка с открытым замком).
5. В открывшемся окне введите слово `Bearer`, **поставьте пробел** и вставьте скопированный токен. Примерно так:
   ```
   Bearer eyJhbGciOiJIUzI...
   ```
6. Нажмите **"Authorize"** и закройте окно. 
7. Теперь все запросы будут автоматически отправлять ваш JWT токен, и защищенные методы вернут успех (200 OK) вместо ошибки прав доступа (401 Unauthorized).

## 🛑 Как остановить сервер

- В консоли/терминале: просто нажмите комбинацию **`Ctrl + C`**.
- В IDE: нажмите красную квадратную кнопку **Stop**.
